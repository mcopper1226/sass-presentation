What is React?

A declarative, efficient, and flexible JavaScript library for building complex (or simple) UIs

Open-source and extremely well-supported by a huge community of developers

The React Paradigm

Composable

Build encapsulated components that manage their own state, then compose them together to make complex UIs

Declarative

You design (declare) views for each state in your application and React will handle the DOM manipulation for you; no need to manage how and when DOM manipulation should take place

Virtual DOM

React creates a JavaScript copy of the actual DOM and all computations are performed here; when changes are detected on the virtual copy, ONLY what has changed is updated in the HTML DOM

One-Way Data Flow

Application data flows one-way via the state and props objects; in a multicomponent hierarchy, a common parent component should manage state and pass it down

Core Concepts

Elements

The smallest building block; they describe what you want to see on the screen and are immutable

							
								$magenta: #d33682;
								$cyan: #35c9be;
								$font_lg: 24px;
								$primary_font: 'Roboto', sans-serif;

								$base_color: $magenta;
								$text_color: $cyan;

								h3 {
									color: $cyan;
									background-color: $base_color;
									font-size: $font-lg;
									font-family: $primary_font;
								}

Components

A collection of HTML, CSS, JS, and internal data specific to that component; each should be independent and focused on doing one specific thing

							
								.image-grid {
									width: 80%;
									padding: 1rem;
									img {
										display: block;
										width: 25%;
										height: auto;
									}
									.text-wrapper {
										margin: 1rem 0;
										h2 {
											color: green;
											&:hover {
												color: orange;
											}
										}
										&:after {
											content: 'Hi!';
										}
									}
								}

Props

Immutable parameters passed into a component from a parent component

State

The internal data store (object) of a component; if data is going to change, we have to use state (not props)

Lifecycle Methods

These allow us to excute certain actions at particular times during the lifecycle of a compoent (mount, update, unmount)

Universal (Server-Side) Rendering

Generate static markup on the server, which is ready to be rendered in the browser, as-is; this is important for speed and SEO

Feature 5: Extends

Base a CSS rule on another rule

							
								.button {
									height: 40px;
									line-height: 40px;
									border-radius: 3px;
								}
								.button-solid {
									@extend .button;
									background: green;
									color: yellow;
								}

Feature 6: Mixins

A groups of CSS declarations that can be reused throughout the site

							
						@mixin backImage($image, $position: center, $size: cover) {
						    background-position: $position;
						    background-size: $size;
						    background-repeat: no-repeat;
						    background-image: url($image);
						}

						.home-hero {
							@include backImage('http://unsplash.com/image-1.jpg', center, cover);
						}
